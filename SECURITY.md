# Security Policy

## Reporting a Vulnerability

If you discover a security vulnerability in SEOGARD, please report it responsibly.

**Email:** support@seogard.io

**Do NOT open a public GitHub issue for security vulnerabilities.**

## What to Include

- Description of the vulnerability
- Steps to reproduce
- Potential impact
- Suggested fix (if any)

## Response Timeline

- **Acknowledgment:** within 48 hours
- **Initial assessment:** within 5 business days
- **Fix or mitigation:** as soon as reasonably possible, depending on severity

## Scope

**In scope:**
- SEOGARD application code (this repository)
- Authentication and authorization logic
- Data exposure or leakage
- Injection vulnerabilities (XSS, NoSQL injection, etc.)
- Server-side request forgery (SSRF)
- Crawler security issues

**Out of scope:**
- Third-party services (Stripe, Resend, etc.)
- Social engineering attacks
- Denial of service (DoS) without a specific vulnerability
- Issues in dependencies (report those upstream, but let us know)

## Disclosure

We will coordinate with you on disclosure timing. We aim to release a fix before any public disclosure.

Thank you for helping keep SEOGARD secure.
